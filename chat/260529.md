## UX 设计 — Step 7：核心体验定义（体验力学）

深入定义 `big commit` 的体验力学：

**定义性体验**：`big commit`——一行命令锁定完整快照
**用户心智模型**：状态快照（非文件变更历史），SVN 式集中式操作期望
**成功标准**：提交即锁定、路径零变化、配方自动记录、反馈即时明确、渐进式披露

**新颖性模式分析：**
- 已有模式：commit/checkout 双命令循环（直接采纳 SVN）
- 组合创新：配方即缓存 Key、生命周期四色编码
- 新颖模式：血缘图跨分支链接（"决策族谱"隐喻）、PPA 排名驱动淘汰（"自动保洁员"隐喻）

**体验力学四步流程**：发起→交互（异步哈希+进度条）→反馈（✓/✗ + 摘要/原因）→完成（工具继续运行）

## UX 设计 — Step 10：用户旅程流程

从 PRD 6 个旅程中识别 5 个关键流程并设计：

1. **commit 提交流程** — 变更检测→异步哈希→输入校验→配方记录→CAS写入→✓反馈
2. **lineage 追溯流程** — 血缘查询→深度检测(>5层截断)→决策链摘要→按需展开
3. **checkout 回退流程** — 未提交检测→CAS构建→大规模提示(--subset)→硬链接替换→✓反馈
4. **promote 晋升流程** — 阶段显示→校验→Golden二次确认→标签更新+策略切换→✓+动画
5. **跨分支依赖预警** — 依赖检测→下游标记→预警通知(非阻塞)→持久化等待确认

提取 5 个共性模式：安全网检查、渐进式披露、确定性反馈、异步非阻塞、预防性验证

4 条优化原则：最少步骤、决策点最小化、进度可见、错误可操作

## UX 设计 — Step 12：UX 一致性模式

定义 6 类一致性模式：

1. **反馈模式** — ✓/✗/⚠/ℹ + Toast/Alert/Modal/Notification，成功≤3行，失败必含建议
2. **生命周期标记** — CLI `[Candidate]` 颜色标签 + GUI Badge，三重传达（颜色+文字+图标）
3. **导航与定位** — CLI ID = GUI 搜索锚点，Ctrl+K 命令面板，URL 状态管理，悬停显示等效 CLI 命令
4. **破坏性操作确认** — 可逆不确认，不可逆必须确认，Golden 需输入确认文本，默认取消(y/N)
5. **搜索与筛选** — CLI 默认20条+--limit，GUI 模糊匹配实时高亮，索引搜索
6. **加载与空状态** — 空状态必有引导+CTA，加载>2秒显示进度，骨架屏匹配布局

**设计系统集成规则：** Token 一致性、8px 网格间距守恒、CLI/GUI 语义色统一、Ant Design Icons 优先、组件优先级（原生>组合>自定义）

## UX 设计 — Step 13：响应式设计与无障碍

**响应式策略：** 仅桌面端，无移动/平板

| 场景 | 策略 |
|------|------|
| 全高清 1920x1080+ | 三栏全展开 |
| 笔记本 1366x768 | 紧凑布局 |
| SSH 终端 | CLI 文本自适应 |
| 多显示器 | Electron 多窗口 |

**断点（桌面优先）：**
- compact ≤1280px — 左侧可折叠为图标栏
- standard 1281-1919px — 三栏可见
- wide ≥1920px — 左侧320px，中央可并排

**无障碍目标：WCAG 2.1 Level AA**
- 文本对比度 ≥ 4.5:1
- 键盘全覆盖 Tab/Enter/Esc/方向键
- ARIA labels + role + aria-live
- 三重传达（颜色+文字+图标）
- 2px 青蓝焦点环
- prefers-reduced-motion 关闭过渡
- 200% 缩放不崩坏
- Golden 色(#FADB14)搭深色文字(#874D00)
- G6 图谱叠加隐藏语义 DOM

**测试：** axe-cli + 键盘遍历 + NVDA/VoiceOver + 色盲模拟 + WebAIM 对比度验证
