## UX 设计 — Step 3：核心体验定义

定义 BIG 的核心体验循环 **commit → explore → checkout**：

- **commit**：一条命令锁定完整快照，快、简单、可靠
- **explore**：在版本历史和血缘图中导航，CLI 快速查询 + GUI 全局可视化
- **checkout**：切换版本，路径不变，EDA 工具无感知，"时间旅行"

**定义性体验**：`big commit`——如果只做好一个交互，这就是一切的基础。

**平台策略**：CLI 主战场(~70%)、GUI 望远镜(~25%)、Python API 系统集成层(~5%)

**体验原则**：零侵入、最少认知负荷、渐进披露、界面即工具、数据驱动信任

用户确认"准确，继续"。

## UX 设计 — Step 4：期望情感响应

核心情感目标：**从焦虑到安心（From Anxiety to Calm Confidence）**

- 情感旅途：怀疑→惊喜→顺手→释然→自信→放心→从容
- 微情感映射：信心 vs 怀疑、安心 vs 焦虑、掌控感 vs 无力感、成就感 vs 挫败感、信任感 vs 不信任
- 设计启示：确定性反馈、预防优于补救、透明可见、渐进成就
- 情感设计原则：确定性反馈、预防优于补救、透明可见、渐进成就

## UX 设计 — Step 5：UX 模式分析与灵感来源

分析 5 个参考产品的可借鉴模式：

1. **SVN** — 双命令心智模型（commit/update），BIG 直接采纳
2. **DVC** — 配方即缓存（Run Cache），配方既管存储又管缓存策略
3. **Perforce** — Lazy Copy 零存储分支 + Workspace 按需同步
4. **W&B/MLflow** — 参数·指标·产物三维绑定 + 运行分组
5. **VS Code** — 三栏布局 + 命令面板 + 渐进披露

**反模式清单**：Git 三区模型、双系统认知负担（DVC+Git）、全量 checkout 默认行为、沉默操作、信息过载默认展示、GUI 功能对称 CLI

**采纳策略**：直接采纳（SVN 双命令、VS Code 三栏、四色编码）；适配改造（W&B 分组→DSO、DVC Run Cache→制品集粒度、Perforce Workspace→--subset）；坚决避免（Git 三区、双系统依赖、全量加载、沉默操作）

用户确认"这些工具足够"。

## UX 设计 — Step 7：核心体验定义（体验力学）

深入定义 `big commit` 的体验力学：

**定义性体验**：`big commit`——一行命令锁定完整快照
**用户心智模型**：状态快照（非文件变更历史），SVN 式集中式操作期望
**成功标准**：提交即锁定、路径零变化、配方自动记录、反馈即时明确、渐进式披露

**新颖性模式分析：**
- 已有模式：commit/checkout 双命令循环（直接采纳 SVN）
- 组合创新：配方即缓存 Key、生命周期四色编码
- 新颖模式：血缘图跨分支链接（"决策族谱"隐喻）、PPA 排名驱动淘汰（"自动保洁员"隐喻）

**体验力学四步流程**：发起→交互（异步哈希+进度条）→反馈（✓/✗ + 摘要/原因）→完成（工具继续运行）

## UX 设计 — Step 8：视觉设计基础

**主色调**：深青蓝（#0E7C86）——区别于 Ant Design 默认蓝，传递专业性与安稳感

**语义色彩**：Primary #0E7C86 / Success #52C41A / Warning #FA8C16 / Error #F5222D / Info #1677FF

**生命周期四色系统（CLI + GUI 统一）：**
| 阶段 | 色值 | CLI | GUI |
|------|------|-----|-----|
| Exploring | #8C8C8C | 灰色标签 | 灰色徽章 |
| Candidate | #1677FF | 蓝色标签 | 蓝色徽章 |
| Pinned | #FA8C16 | 橙色标签 | 橙色徽章 |
| Golden | #FADB14 + #D4B106 描边 | 黄色标签 | 金色徽章 + 星标 |

**字体系统**：中文 Noto Sans SC / 英文 Inter / 代码 JetBrains Mono
**间距**：8px 基准网格（xs 4 / sm 8 / md 16 / lg 24 / xl 32 / xxl 48）
**布局原则**：信息密度优先、三栏弹性布局、呼吸感靠结构非留白、CLI 输出对齐

**无障碍**：WCAG AA 对比度、颜色+文字+图标三重传达、全键盘操作、200% 缩放、prefers-reduced-motion
